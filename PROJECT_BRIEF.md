## Content priorities

## Visual direction

Describe the desired feeling in plain language. Examples:

- quiet editorial field guide;
- warm personal portfolio;
- precise technical reference;
- playful small-product landing page.

## Constraints

- Preserve semantic HTML and keyboard accessibility.
- Keep the project usable without JavaScript wherever practical.
- Build identity in `site.css` rather than rewriting the foundation without a reason.
- Remove placeholder content instead of merely adding around it.

## Definition of done

- The title and metadata describe the actual project.
- The page has one clear primary purpose.
- Placeholder copy is gone.
- Images have useful alternative text.
- Desktop and mobile layouts feel intentional.
- `npm test` passes.
